setTimeout and setInterval in JavaScript

setTimeout and setInterval are special apis in web because they provide a way to schedule code execution asynchronously in JavaScript, which is inherently single-threaded and non-blocking.

or

In other words, setTimeout and setInterval are JavaScript functions that allow you to schedule the execution of code after a certain delay.

setTimeout

The setTimeout function is used to execute a specified function or code snippet after a defined delay.

Syntax


setTimeout ( functionReference/functionDefination, delay, ...args );
  1. function Reference/Definition: This is the function that you want to execute after the specified delay. It can be either a reference to an existing function or an inline function definition.
  2. delay: This is the delay in milliseconds before the function is executed.
  3. args: These are optional additional arguments that can be passed to the function when it is executed. You can pass any number of arguments as needed.

Return

It returns the timer ID.

Example : 


setTimeout(() => {
    console.log('Delayed function executed after 2 seconds');
}, 2000);

Now the statement will be consoled after 2 seconds of delay.

Behind the scenes, when setTimeout is called, the browser schedules the execution of the provided function to occur after the specified delay. It doesn't block the main thread, allowing other tasks to continue while waiting for the timeout to complete.

setTimeout accepts two arguments: the function to execute and the delay in milliseconds. Optionally, additional arguments can be passed to the function. It returns a unique identifier (timer ID) that can be used to cancel the timeout using the clearTimeout function.

setInterval

The setInterval function is similar to setTimeout but is used to repeatedly execute a specified function or code snippet at fixed intervals.

Syntax


setInterval ( functionReference/functionDefination, delay, ...args );

Return

It returns the timer ID.

Example : 


let counter = 0;
let intervalId = setInterval(() => {
    counter++;
    console.log('Interval function executed:', counter);
}, 1000);

Behind the scenes, setInterval works by scheduling the execution of the provided function at regular intervals, defined by the specified time interval. It continues to execute the function indefinitely until clearInterval is called or the page is unloaded.

setInterval accepts two arguments: the function to execute and the interval in milliseconds. Like setTimeout, additional arguments can be passed to the function. It returns a unique identifier (timer ID) that can be used to cancel the interval using the clearInterval function.

clearTimeout

The clearTimeout function is used to cancel a timeout that was set using the setTimeout function. It takes the timer ID returned by setTimeout as an argument and cancels the scheduled timeout.The clearTimeout() function is used to stop or clear a timer set with the setTimeout() method.

Syntax


clearTimeout ( timerID );

Example : 


// Example using setTimeout
const time = setTimeout(() => {
    console.log('Delayed function executed after 2 seconds');
}, 2000);

clearTimeout(time);

clearInterval

The clearInterval function is used to cancel a repeated action that was set using the setInterval function. It takes the timer ID returned by setInterval as an argument and cancels the scheduled interval. The clearInterval() function is used to stop or clear a timer set with the setInterval() method.

Syntax


clearInterval ( timerID );

Example : 


let counter = 0;
let intervalId = setInterval(() => {
    counter++;
    console.log('Interval function executed:', counter);
    if (counter === 5) {
        clearInterval(intervalId);
        console.log('Interval stopped after 5 executions');
    }
}, 1000);

setImmediate

The setImmediate function is used to execute a specified function or code snippet after the current execution stack is cleared.

Syntax


setImmediate ( functionReference/functionDefination, ...args );

Return

It returns the timer ID.

Example : 


setImmediate(() => {
    console.log('Immediate function executed');
});

clearImmediate

The clearImmediate function is used to cancel a timeout that was set using the setImmediate function.

Syntax


clearImmediate(immediateID);

Example : 


const immediate = setImmediate(() => {
    console.log('Immediate function executed');
});

setTimeout(() => {
    clearImmediate(immediate);
    console.log('Immediate stopped');
}, 1000);

Working of setTimeout/Interval/Immediate methods

  • When you call setTimeout, you're essentially telling the browser: "Execute this code after a certain amount of time."
  • The browser registers the callback function (the function you want to execute after the delay) and the specified delay time.
  • It then continues executing the rest of your code.
  • After the specified delay has elapsed, the browser puts the callback function into the task queue.
  • When the main thread becomes available (i.e., it's not busy executing other tasks), it pulls the callback function from the event queue, make it pass through Event loop and executes it.

It's important to note that JavaScript, being single-threaded, can only execute one piece of code at a time. So, while setTimeout and setInterval allow you to defer code execution, they don't create additional threads or parallelism. Instead, they leverage event-driven architecture to handle asynchronous behavior.

Conclusion

setTimeout and setInterval are powerful tools in JavaScript for managing asynchronous code execution and scheduling time-based operations. Understanding their behavior and usage is essential for developing efficient and responsive web applications.
