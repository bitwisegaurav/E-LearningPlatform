Async and Await in JavaScript

Async functions and the await keyword provide a more concise and synchronous-like syntax for working with promises in JavaScript. They allow you to write asynchronous code that looks and behaves like synchronous code, making it easier to understand and maintain.

1. Async Functions

An async function is a function declared with the async keyword. It allows you to define asynchronous operations within the function body using the await keyword. Async functions always return a promise, which resolves with the function's return value or rejects with an error thrown within the function.

2. Await Keyword

The await keyword can only be used within an async function. It pauses the execution of the async function until the awaited promise is resolved or rejected. This allows you to write asynchronous code that appears synchronous and follows the natural flow of control.

3. Error Handling

Async functions and await simplify error handling compared to traditional promise-based code. You can use try-catch blocks to handle errors synchronously within the async function, making it easier to manage and propagate errors.

4. Example:

// Example of using async function and await keyword
async function fetchData() {
    try {
        const data = await fetch('https://api.example.com/data');
        const json = await data.json();
        console.log(json);
        return json;
    } catch (error) {
        console.error('Error fetching data:', error);
        throw error;
    }
}

// Call async function
fetchData()
    .then((data) => {
        console.log('Data fetched successfully:', data);
    })
    .catch((error) => {
        console.error('Error:', error);
    });

Conclusion

Async and await provide a more intuitive and readable way to work with asynchronous code in JavaScript. By using async functions and the await keyword, developers can write asynchronous operations that resemble synchronous code, leading to cleaner, more maintainable, and error-resistant code.
