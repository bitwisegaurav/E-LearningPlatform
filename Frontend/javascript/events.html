Events

Events are actions or occurrences that happen in the browser as a result of user interaction or other triggers.

Examples of events include mouse clicks, keyboard inputs, form submissions, page loads, and changes in the state of HTML elements. Events provide a way for JavaScript to respond to user interactions and trigger actions or behaviors in the web application.

Event Object

When an event occurs, an event object is created and passed as an argument to the event listener function. This event object contains information about the event, such as the type of event, the target element, and any additional data related to the event. Event objects allow developers to access and manipulate event-related data within event listener functions.

Event Listeners

Event listeners are functions that are attached to specific DOM elements and wait for particular events to occur.

Once the event occurs, the associated event listener is triggered, and the defined action or behavior is executed.

Adding Event Listeners

Event listeners can be added to DOM elements using JavaScript. We can add event listeners by 3 methods :

Here we are taking an example in which a alert will be pop up when the button is clicked.

Inline onclick

The onclick attribute is added to the element and assigned a JavaScript function to be executed when the event occurs.

<button onclick="alert('Button clicked!');">Click Me<button>

onclick property

The onclick property is added to the element and assigned a JavaScript function to be executed when the event occurs.

<button id="myButton">Click Me<button>

let button = document.getElementById('myButton');
button.onlcick = function() {
    alert('Button Clicked!');
}

addEventListener method

The addEventListener method accepts three arguments: the event type, the event listener function, and an optional third parameter that specifies whether the event should be captured during the event propagation phase.

let button = document.getElementById('myButton');
button.addEventListener('click', function() {
    console.log('Button clicked!');
});

Result :

The third argument in the addEventListener method is used to specify whether the event should be captured or bubbled.

During the capturing phase, the event starts from the outermost element and moves towards the target element. This allows you to handle the event on parent elements before it reaches the target element.

During the bubbling phase, the event starts from the target element and moves towards the outermost element. This allows you to handle the event on the target element and then on its parent elements.

By default, the third argument of addEventListener is set to false, which means the event will be handled during the bubbling phase.

Removing Event Listeners

The removeEventListener method in JavaScript is used to remove event handlers that have been attached with the addEventListener method.

For example : 

// Define the function
function handleClick(event) {
    console.log('Button clicked');
}

// Get the button element
let button = document.getElementById('myButton');

// Add the event listener
button.addEventListener('click', handleClick);

// Later in your code, you can remove the event listener
button.removeEventListener('dblclick', handleClick);

Conclusion

Events and event listeners are fundamental concepts in web development, enabling developers to create interactive and dynamic user experiences in web applications. By understanding how to use events and event listeners effectively, developers can build applications that respond dynamically to user interactions and provide a rich and engaging user experience.
