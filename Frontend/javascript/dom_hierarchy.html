DOM Hierarchy Fundamentals

In JavaScript, you can easily traverse the DOM tree to access an element's children, parent, and siblings. These relationships helps us in manipulating the structure of a web page dynamically.

Children of an Element

The children property of an element returns a collection of all child elements of the specified element. Child elements are the elements that are directly nested within the parent element.

This property excludes text nodes and comment nodes. You can iterate over this collection to access and manipulate each child element individually.

Example : 


let parentElement = document.getElementById('parent');
let children = parentElement.children;
for (let i = 0; i < children.length; i++) {
    console.log(children[i]);
}

Some properties/methods :

  • childElementCount : Number of child elements directly nested within the parent element.
  • hasChildNodes : Checks if the element has any child nodes, including text nodes and comment nodes.
  • children : HTMLCollection of all child elements of the specified element.
  • childNodes : Return NodeList of all child nodes of the specified element, including text nodes and comment nodes.
  • firstChild : First child node of the specified element, including text nodes and comment nodes.
  • lastChild : Last child node of the specified element, including text nodes and comment nodes.
  • firstElementChild : First child element of the specified element.
  • lastElementChild : Last child element of the specified element.
  • appendChild : Appends a node as the last child (as by default) of the specified element.
  • removeChild : Removes a child node from the specified element.
  • replaceChild : Replaces a child node of the specified element with a new node.

Parent of an Element

The parentElement property of an element returns the parent element of the specified element. This property provides direct access to the parent element, allowing you to traverse up the DOM tree.

Example : 


let childElement = document.getElementById('child');
let parent = childElement.parentElement;
console.log(parent);

Some properties/methods :

  • parentElement : parentElement property returns the Parent element of the specified element.
  • parentNode : parentNode property returns the parent node of the specified element, including text nodes and comment nodes.

Siblings of an Element

The previousElementSibling and nextElementSibling properties of an element return the previous and next sibling elements, respectively. These properties enable you to navigate horizontally within the DOM tree, accessing adjacent elements.

Example : 


let sibling = childElement.nextElementSibling;
console.log(sibling);

Some properties/methods :

  • previousSibling: Returns the previous sibling node, including text and comment nodes.
  • nextSibling: Returns the next sibling node, including text and comment nodes.
  • insertAdjacentElement(position, element): Inserts a provided element into the DOM relative to the current element, at a specified position (e.g., 'beforebegin', 'afterbegin', 'beforeend', 'afterend').
  • insertAdjacentHTML(position, markup): Inserts a provided string of HTML or XML into the DOM relative to the current element, at a specified position (e.g., 'beforebegin', 'afterbegin', 'beforeend', 'afterend').
  • insertAdjacentText(position, text): Inserts a provided string of text into the DOM relative to the current element, at a specified position (e.g., 'beforebegin', 'afterbegin', 'beforeend', 'afterend').
  • insertBefore(newNode, referenceNode): Inserts a new node before a specified existing node, as a child of the current node's parent.
  • replaceChild(newNode, oldNode): Replaces a child node with a new node within the current node's parent

Conclusion

Understanding the relationships between elements in the DOM tree, including children, parent, and siblings, is essential for manipulating the structure of a web page dynamically. By leveraging properties such as children, parentElement, previousElementSibling, and nextElementSibling, developers can easily navigate and interact with elements in the DOM.
