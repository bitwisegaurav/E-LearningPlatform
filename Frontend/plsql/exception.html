Exception Handling

PL/SQL provides a powerful mechanism to handle errors and exceptions that occur in a program. Exception handling allows you to handle run-time errors gracefully without terminating the program abruptly. It helps in debugging, tracing errors, and providing meaningful error messages to the user.

Types of Exceptions:

  • Predefined Exceptions : PL/SQL provides several predefined exceptions that are raised when a run-time error occurs. These exceptions are divided into two categories: internally defined and user-defined exceptions.
  • User-Defined Exceptions : PL/SQL allows you to define your own exceptions to handle specific errors that may occur in your program. User-defined exceptions are declared in the declaration section of the PL/SQL block.
  • Unhandled Exceptions : If an exception occurs and there is no exception handler to handle it, the program terminates abnormally, and an error message is displayed.

Syntax :

BEGIN
    -- code block
EXCEPTION
    WHEN exception1 THEN
        -- handler code block
    WHEN exception2 THEN
        -- handler code block
    -- additional exception handlers
END;

In the above syntax, we have a code block followed by an exception block. The exception block contains one or more exception handlers that handle different types of exceptions. Each exception handler is associated with an exception name.
