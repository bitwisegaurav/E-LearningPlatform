Analysis of Algorithms

Introduction

Algorithm analysis is an essential aspect of computer science that involves studying the efficiency and performance characteristics of algorithms. It helps us understand how algorithms behave and make informed decisions about their usage in various applications.

Types of Analysis

There are different types of analyses performed on algorithms:

  1. Time Complexity Analysis: Measures the amount of time an algorithm takes to complete as a function of the input size.
  2. Space Complexity Analysis: Evaluates the amount of memory space an algorithm requires in relation to the input size.

Functions for Computer Science

Floor Function

The floor function of any number means greatest integer which is less than or equal to that number. It is denoted by ⌊x⌋. For example ⌊3.14⌋ = 3.

Ceiling Function

The ceiling function of any number means least integer which is greater than or equal to that number. It is denoted by ⌈x⌉. For example ⌈3.14⌉ = 4.

Integer Function

The integer function is a function that maps any real number to an integer. It is denoted by INT(x). For example INT(3.14) = 3.

Absolute Value Funciton

The absolute value function is a function that maps any real number to its absolute value. It is denoted by |x| or ABS(x). For e.g |-5.1| = 5.1.

Characterstic Function

The characteristic function of a set is a function that maps each element of the set to a boolean value. It is denoted by cS : A → {0,1} be defined by : 

cS(x) = {
    1 if x ∈ S,
    0 if x ∉ S
}

Special Notations for growth functions

Big-O

Big-O notation is a mathematical notation that describes the upper bound or worst-case scenario of the growth rate of a function as the size of the input increases. It provides an estimate of the maximum number of key operations required by an algorithm.

Big-omega(Ω)

Big-Ω notation is a mathematical notation that describes the lower bound or best-case scenario of the growth rate of a function as the size of the input increases. It provides an estimate of the minimum number of key operations required by an algorithm.

Big-Theta(Θ)

Big-Θ notation is a mathematical notation that describes both the upper and lower bounds, providing a tight range of the growth rate of a function as the size of the input increases. It gives a more precise estimate of the number of key operations required by an algorithm.

Growth Rate Function

The time efficiency of almost all of the algorithms can be characterized by only a few growth rate functions :

  1. Constant Time: The growth rate of a function is constant if it is a straight line. It is denoted by O(1).
  2. Linear Time: The growth rate of a function is linear if it is a straight line. It is denoted by O(n).
  3. Quadratic Time: The growth rate of a function is quadratic if it is a parabola with a positive slope. It is denoted by O(n2).
  4. Cubic Growth Rate Function: The growth rate of a function is cubic if it is a parabola with a positive slope. It is denoted by O(n3).
  5. Exponential Time: The growth rate of a function is exponential if it is a parabola with a negative slope. It is denoted by O(2n).
  6. Logarithmic Time: The growth rate of a function is logarithmic if it is a parabola with a negative slope. It is denoted by O(log n).
  7. Factorial Time: The growth rate of a function is factorial if it is a parabola with a negative slope. It is denoted by O(n!).
  8. Polynomial Growth Rate Function: The growth rate of a function is polynomial if it is a parabola with a negative slope. It is denoted by O(nk).

Conclusion

Algorithm analysis is crucial for making informed decisions in software development, choosing the right algorithms for specific tasks, and optimizing the overall performance of computer systems.
