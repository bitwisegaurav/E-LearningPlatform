Multithreading in Java

Multithreading is a feature in Java that allows concurrent execution of two or more threads. A thread is a lightweight process, and multithreading is a way to improve the performance of your programs by executing multiple tasks concurrently.

Main is also known as parent thread because all execution starts from their and all other threads are child threads of main thread.

Qualified name of thread class is java.lang.Thread.

Creating and Managing Threads

In Java, there are two ways to create threads:

  1. By implementing the Runnable interface :

    If we user Runnable interface, then we have to override a function, that is run function. Whenever a thread is executed, then it always execute the run function.

    To override the run function, we will use the following syntax : 

    public void run() {

    For example :

    class MyRunnable implements Runnable {
    public void run() {
        // Code to be executed in the new thread
    }
}

// Creating and starting a thread
Thread thread = new Thread(new MyRunnable());
thread.start();
  2. By extending the Thread class :

    If we implements Runnable interface then we have to create threads using Thread class which can takes objects of a class which implements Runnable as argunments.

    And to run a thread, we used to call a function known as start. And we cannot call run function directly.

    For example :

    class MyThread extends Thread {
    public void run() {
        // Code to be executed in the new thread
    }
}

// Creating and starting a thread
MyThread thread = new MyThread();
thread.start();

Commonly Used Functions in Multithreading

1. start()

The start() method is used to begin the execution of a thread. It invokes the run() method on the thread, and the new thread begins its life cycle. 

MyThread thread = new MyThread();
thread.start();

2. join()

The join() method is used to wait for a thread to finish its execution before moving on to the next step. It ensures that the current thread waits until the specified thread completes. 

Thread thread = new Thread(new MyRunnable());
thread.start();
try {
    thread.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}

3. sleep()

The sleep() method is used to pause the execution of a thread for a specified duration. It allows other threads to run during the sleep period. 

try {
    Thread.sleep(1000); // Sleep for 1 second
} catch (InterruptedException e) {
    e.printStackTrace();
}

4. yield()

The yield() method is used to temporarily pause the execution of the currently running thread, allowing other threads of the same priority to run. 

Thread.yield();

5. isAlive()

The isAlive() method is used to check whether a thread is still alive (has started and not yet terminated) or not. 

Thread thread = new Thread(new MyRunnable());
thread.start();
boolean aliveStatus = thread.isAlive();

6. setPriority()

The setPriority() method is used to set the priority of a thread. Threads with higher priority get more preference in execution. 

MyThread highPriorityThread = new MyThread();
highPriorityThread.setPriority(Thread.MAX_PRIORITY);

MyThread lowPriorityThread = new MyThread();
lowPriorityThread.setPriority(Thread.MIN_PRIORITY);

Synchronization

In multithreading, synchronization is crucial to ensure that threads don't interfere with each other. The synchronized keyword is used to achieve synchronization. 

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

Conclusion

Multithreading is a powerful feature in Java that enables concurrent execution. Understanding how to create and manage threads, along with synchronization techniques, is essential for building efficient and responsive applications.
